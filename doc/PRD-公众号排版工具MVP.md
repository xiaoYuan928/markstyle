# 产品需求文档（PRD）

# 公众号排版工具 MVP

> 版本：v1.0
> 日期：2025-12-18
> 基于：Doocs/md 开源项目二次开发

---

## 1. 产品概述

### 1.1 产品定位

**"公众号排版最后一公里"** — 不是编辑器，是美化器

### 1.2 目标用户

- 公众号运营者
- 内容创作者
- 技术博主
- 自媒体从业者

### 1.3 用户痛点（基于竞品分析）

| 痛点     | 现有解决方案的问题                                     |
| -------- | ------------------------------------------------------ |
| 排版耗时 | 飞书/Notion → Markdown编辑器 → 墨滴 → 公众号，流程太长 |
| 主题太少 | Doocs/md 只有 2 个内置主题                             |
| 付费门槛 | 墨滴好看的主题要付费（¥98-198/年）                     |
| 功能太重 | 现有工具都是"编辑器"，用户只需要"美化"                 |

### 1.4 核心价值主张

- **免费**：所有主题完全免费
- **好看**：提供 15-20 个精美主题
- **简单**：粘贴即美化，复制即走
- **轻量**：无需注册，即开即用

## 3. MVP 核心功能

### 3.1 功能优先级矩阵

| 优先级 | 功能             | MVP必须 | 说明                      |
| ------ | ---------------- | ------- | ------------------------- |
| P0     | Markdown 输入    | ✅      | 继承 Doocs/md             |
| P0     | 实时预览         | ✅      | 继承 Doocs/md             |
| P0     | 一键复制到公众号 | ✅      | 继承 Doocs/md             |
| P0     | 主题切换         | ✅      | **重点改造：增加主题**    |
| P1     | 富文本粘贴转换   | ✅      | 从飞书/Notion粘贴自动转MD |
| P1     | 代码块主题       | ✅      | 继承 Doocs/md             |
| P2     | 自定义主题色     | ⚪      | 后续迭代                  |
| P2     | 本地历史记录     | ⚪      | 后续迭代                  |
| P3     | 用户自制主题     | ⚪      | 后续迭代                  |
| P3     | 主题市场         | ⚪      | 后续迭代                  |

### 3.2 MVP 核心功能详述

#### 功能1：主题库（核心差异化）

**目标：** 提供 15-20 个免费精美主题，比 Doocs 多，比墨滴免费

**主题分类：**

```
├── 简约风格（5个）
│   ├── 极简白
│   ├── 墨黑
│   ├── 灰度
│   ├── 淡雅
│   └── 清新
├── 科技风格（4个）
│   ├── 程序员
│   ├── 暗夜码农
│   ├── GitHub风
│   └── 终端绿
├── 商务风格（3个）
│   ├── 专业蓝
│   ├── 企业红
│   └── 金融灰
├── 创意风格（4个）
│   ├── 渐变彩虹
│   ├── 橙黄活力
│   ├── 粉紫少女
│   └── 森林绿
└── 节日风格（4个）
    ├── 春节红
    ├── 圣诞绿
    ├── 万圣节
    └── 情人节
```

**验收标准：**

- [ ] 至少 15 个主题可用
- [ ] 主题切换实时预览
- [ ] 主题支持：标题、正文、引用、代码块、表格、列表

#### 功能2：粘贴即美化

**用户流程：**

```
1. 用户在飞书/Notion/其他工具写好文章
2. 复制内容
3. 粘贴到我们的工具
4. 自动识别格式（Markdown 或富文本）
5. 选择主题
6. 一键复制到公众号
```

**验收标准：**

- [ ] 支持 Markdown 格式粘贴
- [ ] 支持富文本粘贴自动转 Markdown（P1）
- [ ] 粘贴后 500ms 内完成渲染

#### 功能3：一键复制

**验收标准：**

- [ ] 复制后可直接粘贴到公众号编辑器
- [ ] 保留所有样式（字体、颜色、间距）
- [ ] 图片正常显示（需用户先上传到图床）

## 4. 技术方案

### 4.1 技术选型

基于 Doocs/md 开源项目：

- **前端框架**：next.js
- **样式**：TailwindCSS
- **Markdown 解析**：markdown-it
- **代码高亮**：highlight.js
- **部署**：Vercel / Netlify（免费）

### 4.2 二次开发重点

| 模块     | 改动程度 | 说明                   |
| -------- | -------- | ---------------------- |
| 主题系统 | **大改** | 新增 15+ 主题 CSS      |
| UI 界面  | 中改     | 简化界面，突出主题选择 |
| 品牌元素 | 小改     | Logo、名称、页脚       |
| 核心功能 | 不改     | 继承 MD 解析和复制功能 |

### 4.3 项目结构（关键文件）

```
doocs-md/
├── src/
│   ├── assets/
│   │   └── themes/          # 【重点】主题 CSS 文件
│   │       ├── default.css
│   │       ├── elegant.css
│   │       └── ... (新增主题)
│   ├── components/
│   │   └── ThemeSelector.vue # 【重点】主题选择器组件
│   ├── config/
│   │   └── themes.js        # 【重点】主题配置
│   └── App.vue
├── public/
│   └── favicon.ico          # 替换 Logo
└── index.html               # 修改标题
```

## 5. 部署方案

### 5.1 推荐方案：Vercel（免费）

**优势：**

- 完全免费
- 自动 CI/CD（push 即部署）
- 全球 CDN
- 支持自定义域名
- 自动 HTTPS

**步骤：**

1. Fork Doocs/md 到自己的 GitHub
2. 在 Vercel 导入项目
3. 自动部署完成
4. 绑定自定义域名

### 5.2 备选方案

| 方案         | 费用    | 适合场景   |
| ------------ | ------- | ---------- |
| Netlify      | 免费    | 静态站点   |
| GitHub Pages | 免费    | 简单部署   |
| 阿里云 OSS   | ~¥10/月 | 国内访问快 |
| 腾讯云 COS   | ~¥10/月 | 国内访问快 |
